O simulador interativo e funcional para o módulo Q-CyPIA foi desenvolvido com sucesso! Ele inclui um frontend React e um backend FastAPI, ambos configurados para deploy na Render.
Você pode encontrar todos os arquivos do projeto no diretório /home/ubuntu/qcypia-simulator/.
Para visualizar e testar o simulador localmente:
Frontend:
Bash
cd /home/ubuntu/qcypia-simulator
pnpm install
pnpm run dev
Acesse http://localhost:5173 no seu navegador.
Backend:
Bash
cd /home/ubuntu/qcypia-simulator/backend
pip install -r requirements.txt
uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
A API estará disponível em http://localhost:8000 e a documentação interativa em http://localhost:8000/docs.
Para deploy na Render:
Consulte o README.md principal e o backend/README.md para instruções detalhadas sobre como fazer o deploy do frontend e do backend na Render. Lembre-se de configurar as variáveis de ambiente (VITE_API_URL no frontend e ALLOWED_ORIGINS no backend ) após o deploy para que se comuniquem corretamente.
Os arquivos criados são:
/home/ubuntu/qcypia-simulator/ (diretório raiz do projeto)
README.md (README principal com instruções)
.gitignore
.env.example (para o frontend)
src/App.jsx (componente principal do simulador React)
src/App.css (estilos)
index.html
package.json, pnpm-lock.yaml, vite.config.js (configurações do React)
backend/ (diretório do backend FastAPI)
app/main.py (lógica da API)
app/__init__.py
requirements.txt
render.yaml (configuração de deploy para Render)
Dockerfile
.env.example (para o backend)
README.md (README do backend)
Por favor, me avise se precisar de mais alguma coisa ou tiver dúvidas!
