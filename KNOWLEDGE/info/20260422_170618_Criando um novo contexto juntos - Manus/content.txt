Diagnóstico Final e Irrefutável
O problema não está mais no código Java nem na sintaxe do build.gradle. O problema está no processo de build em si. O Android Gradle Plugin, por alguma razão no seu ambiente, não está gerando as classes R.java e BuildConfig.java antes da etapa de compilação do Java. Quando o javac (o compilador Java) vai compilar a MainActivity e o StatusFragment, ele não encontra essas classes que deveriam ter sido criadas.
A causa mais comum para isso, depois de tudo que já tentamos, é uma incompatibilidade entre a versão do Android Gradle Plugin (AGP) e a versão do Gradle.
Ação: Um Passo de Cada Vez - Sincronizar as Versões

Passo Único e Isolado: Corrigir as Versões do Build
Vamos editar apenas um arquivo: o build.gradle do projeto (o que está na raiz, não o que está na pasta app).
Ação:
Abra o arquivo build.gradle que está na raiz do seu projeto.
Substitua todo o conteúdo dele pelo código abaixo. Estamos especificando uma versão estável e comprovada do Android Gradle Plugin.
groovy
// Top-level build file where you can add configuration options common to all sub-projects/modules.
plugins {
    id 'com.android.application' version '8.2.2' apply false
}
O que isso faz?
Isso diz ao Gradle para usar a versão 8.2.2 do Android Gradle Plugin, que é uma versão estável e madura. A versão do Gradle Wrapper (gradlew) irá se ajustar automaticamente para usar uma versão do Gradle compatível com este plugin.

A causa mais comum para isso, depois de tudo que já tentamos, é uma incompatibilidade entre a versão do Android Gradle Plugin (AGP) e a versão do Gradle.
