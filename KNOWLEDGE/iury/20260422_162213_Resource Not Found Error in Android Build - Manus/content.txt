Analisei os arquivos P2PSendReceiveThread.java, TcpSendReceiveThread.java, MainActivity.java e WiFiDirectBroadcastReceiver.java que você me enviou. O método disconnect() está presente e correto nas classes P2PSendReceiveThread e TcpSendReceiveThread.
Isso é uma boa notícia, pois significa que o código que você tem está correto. No entanto, o fato de o erro "cannot find symbol method disconnect()" ainda aparecer na sua captura de tela indica que o problema não está nos seus arquivos, mas sim na forma como o Android Studio está processando ou cacheando esses arquivos. É um problema comum de sincronização ou cache do IDE.
Para resolver isso, vamos tentar uma limpeza e reinicialização mais profunda do Android Studio. Por favor, siga estes passos com atenção:
Feche o Android Studio completamente. Certifique-se de que não há nenhuma instância rodando em segundo plano.
Limpe o cache do Android Studio:
No seu sistema de arquivos, navegue até o diretório de cache do Android Studio. A localização padrão varia um pouco dependendo do sistema operacional:
Windows: C:\Users\SeuUsuario\.AndroidStudio<version>\system\caches
macOS: ~/Library/Caches/Google/AndroidStudio<version>
Linux: ~/.cache/Google/AndroidStudio<version>
Exclua todo o conteúdo da pasta caches. Não exclua a pasta caches em si, apenas o que está dentro dela.
Reinicie o Android Studio. Ele pode demorar um pouco mais para iniciar, pois estará reconstruindo o cache.
Abra o seu projeto MeshWave.
Sincronize o Projeto com os Arquivos Gradle: No Android Studio, vá em File > Sync Project with Gradle Files.
Limpe o Projeto: Vá em Build > Clean Project.
Reconstrua o Projeto: Vá em Build > Rebuild Project.
Este processo forçará o Android Studio a reconstruir todos os índices e caches do projeto, o que geralmente resolve problemas de símbolos não encontrados. Por favor, me informe o resultado após seguir todos esses passos. Se o erro persistir, precisaremos investigar outras possibilidades, mas esta é a etapa mais provável para resolver o problema.

Por favor, tente a seguinte sequência de ações no seu Android Studio, que é o equivalente a um "Rebuild Project" quando a opção direta não está disponível:

Reinicie o Android Studio.
Abra o seu projeto MeshWave.
No Android Studio, vá em Build (no menu superior) e procure por:
Clean Project (execute esta primeira)
Em seguida, procure por Make Project ou Assemble Project (execute uma dessas, se Make Project não estiver disponível, use Assemble Project).
Esses passos combinados forçarão uma limpeza profunda e uma nova construção do projeto, o que deve resolver os problemas de símbolos não encontrados. Por favor, me informe o resultado após tentar essa sequência de ações.

Analisei os arquivos P2PSendReceiveThread.java, TcpSendReceiveThread.java, MainActivity.java e WiFiDirectBroadcastReceiver.java que você me enviou. O método disconnect() está presente e correto nas classes P2PSendReceiveThread e TcpSendReceiveThread.
Isso é uma boa notícia, pois significa que o código que você tem está correto. No entanto, o fato de o erro "cannot find symbol method disconnect()" ainda aparecer na sua captura de tela indica que o problema não está nos seus arquivos, mas sim na forma como o Android Studio está processando ou cacheando esses arquivos. É um problema comum de sincronização ou cache do IDE.
Para resolver isso, vamos tentar uma limpeza e reinicialização mais profunda do Android Studio. Por favor, siga estes passos com atenção:
Feche o Android Studio completamente. Certifique-se de que não há nenhuma instância rodando em segundo plano.
Limpe o cache do Android Studio:
No seu sistema de arquivos, navegue até o diretório de cache do Android Studio. A localização padrão varia um pouco dependendo do sistema operacional:
Windows: C:\Users\SeuUsuario\.AndroidStudio<version>\system\caches
macOS: ~/Library/Caches/Google/AndroidStudio<version>
Linux: ~/.cache/Google/AndroidStudio<version>
Exclua todo o conteúdo da pasta caches. Não exclua a pasta caches em si, apenas o que está dentro dela.
Reinicie o Android Studio. Ele pode demorar um pouco mais para iniciar, pois estará reconstruindo o cache.
Abra o seu projeto MeshWave.
Sincronize o Projeto com os Arquivos Gradle: No Android Studio, vá em File > Sync Project with Gradle Files.
Limpe o Projeto: Vá em Build > Clean Project.
Reconstrua o Projeto: Vá em Build > Rebuild Project.
Este processo forçará o Android Studio a reconstruir todos os índices e caches do projeto, o que geralmente resolve problemas de símbolos não encontrados. Por favor, me informe o resultado após seguir todos esses passos. Se o erro persistir, precisaremos investigar outras possibilidades, mas esta é a etapa mais provável para resolver o problema.
