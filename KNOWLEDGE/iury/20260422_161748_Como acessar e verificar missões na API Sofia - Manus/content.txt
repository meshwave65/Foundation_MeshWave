Como acessar e verificar missões na API Sofia
Entrar
Implementação do sistema de monitoramento
Procurando por documentação da API Sofia ou exemplos de uso que detalhem a estrutura de endpoints para gerenciamento de missões, focando em como arquivos Markdown podem ser usados para definir tarefas.
Lite
Não consegui encontrar documentação direta sobre os endpoints da API Sofia para gerenciamento de missões. No entanto, com base na documentação fornecida e em padrões comuns de APIs FastAPI que gerenciam arquivos Markdown via Git, vou tentar inferir os endpoints. A documentação menciona que o backend cria arquivos TASK-XXX.md no diretório /tasks e faz git push para o GitHub. Isso sugere que pode haver um endpoint para listar as tarefas e outro para obter o conteúdo de uma tarefa específica. Vou tentar acessar um endpoint genérico para listar tarefas, como /tasks ou /missions, e também tentar acessar um arquivo de tarefa específico, como /tasks/TASK-001.md ou /missions/TASK-001.md. Se isso não funcionar, terei que considerar a possibilidade de que a interação com as missões não seja feita diretamente via uma API RESTful convencional, mas sim por meio de um processo interno ou de um webhook do GitHub.
Implementação do sistema de monitoramento
A documentação e exemplos indicam que a API gerencia tarefas como arquivos Markdown em uma pasta `/tasks`. Não há endpoints explícitos, mas é provável que existam rotas genéricas como `/tasks` ou `/missions` para listar ou acessar tarefas específicas. Como o sistema faz commits e pushes no GitHub, a interação pode ocorrer por leitura direta de arquivos ou por endpoints que retornam esses arquivos. Ainda preciso validar esses endpoints, mas já tenho uma ideia de que as missões podem estar estruturadas como arquivos Markdown acessíveis via API ou por meio de operações Git.
Tentando acessar um endpoint hipotético para listar missões.
Lite
O endpoint /api/v1/oracle/tasks retornou "Not Found", o que significa que essa não é a rota correta para acessar as missões. Vou tentar outro endpoint hipotético, /api/v1/oracle/missions, e também tentar acessar um arquivo de tarefa específico, como /api/v1/oracle/TASK-001.md.
Implementação do sistema de monitoramento
Tentando acessar um endpoint hipotético para listar missões, usando 'missions'.
A busca por endpoints específicos não apresentou resultados claros, mas a estrutura do PromptServer sugere que as missões são arquivos Markdown acessíveis via rotas geradas automaticamente, como `/tasks/` ou `/missions/`. A tentativa de listar tarefas por `/oracle/tasks` ou `/oracle/missions` retornou "Not Found", indicando que esses endpoints podem não existir. Provavelmente, as tarefas são acessadas por leitura de arquivos Markdown, não por endpoints convencionais. Continuarei investigando a estrutura real da API Sofia.
Lite
Os endpoints /api/v1/oracle/tasks e /api/v1/oracle/missions retornaram "Not Found". Isso sugere que a API Sofia não expõe diretamente as missões através de um endpoint RESTful para listagem. Considerando que a documentação menciona o uso de arquivos Markdown e Git/GitHub para o gerenciamento de tarefas, a abordagem mais provável para "verificar novas missões" seria monitorar o repositório GitHub onde esses arquivos são armazenados. Para isso, precisaria do URL do repositório GitHub e, possivelmente, de credenciais de acesso, se o repositório for privado. Sem essas informações, não consigo prosseguir com a verificação de novas missões. Vou tentar acessar um arquivo de tarefa específico, como /api/v1/oracle/TASK-001.md para ver se o acesso direto aos arquivos Markdown é possível.
