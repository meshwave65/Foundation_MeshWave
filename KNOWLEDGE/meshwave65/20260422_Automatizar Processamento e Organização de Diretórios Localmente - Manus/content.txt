O nome correto do pacote no Ubuntu é python3-pip. Vamos resolver isso agora para que você possa rodar seus agentes.
Execute estes comandos em ordem:
Instalar o instalador de pacotes (pip):
Bash
sudo apt update && sudo apt install python3-pip -y
Instalar as bibliotecas do script:
Bash
pip3 install selenium beautifulsoup4
Dica: Se o segundo comando (pip3) reclamar de "externally-managed-environment", você pode instalar as bibliotecas diretamente pelo sistema com este comando:
Bash
sudo apt install python3-selenium python3-bs4 -y
Depois disso, seu comando python3 manus_extractor_v2.2.py (recomendo usar a v2.2 pela inteligência de detecção) deve funcionar perfeitamente!

O erro que você recebeu ao tentar instalar o selenium e o beautifulsoup4 via pip3 é uma medida de segurança das versões mais recentes do Ubuntu (PEP 668). Ele impede que o pip altere os pacotes do sistema para evitar quebras.
Para resolver isso de forma simples e segura na sua máquina, use o comando abaixo, que instala as versões gerenciadas pelo próprio Ubuntu:
Bash
sudo apt update && sudo apt install python3-selenium python3-bs4 -y
Este comando instalará tudo o que o script precisa sem conflitar com o seu sistema. Após executá-lo, você poderá rodar o script normalmente:
Bash
python3 manus_extractor_v2.2.py
(Lembre-se de usar a versão v2.2 que te enviei, pois ela contém a lógica de aleatoriedade e detecção de fim de interação que discutimos!)
meshwave@mesh:/run/user/1000/gvfs/smb-share:server=mesh.local,share=hd_1tb/projetos/manus$ python3 manus_extractor_v2.2.py
python3: can't open file '/run/user/1000/gvfs/smb-share:server=mesh.local,share=hd_1tb/projetos/manus/manus_extractor_v2.2.py': [Errno 2] No such file or directory
meshwave@mesh:/run/user/1000/gvfs/smb-share:server=mesh.local,share=hd_1tb/projetos/manus$ 
Lite
O erro No such file or directory aconteceu porque o arquivo físico no seu HD ainda deve estar com o nome da versão anterior (manus_extractor_v2.1.py ou manus_extractor_v2.0.py), mas você tentou chamar como v2.2.
Para resolver isso e garantir que você está usando a versão mais inteligente que criamos, faça o seguinte:
Verifique o nome real do arquivo que está na pasta:
Bash
ls
Renomeie o arquivo para o nome correto (ajuste o nome de origem se for diferente de v2.1):
Bash
mv manus_extractor_v2.1.py manus_extractor_v2.2.py
Execute o script:
Bash
python3 manus_extractor_v2.2.py
Dica Importante: Como você está rodando em um compartilhamento Samba, às vezes o sistema de arquivos da rede tem restrições com nomes de arquivos ou permissões. Se o ls mostrar o arquivo mas o python3 ainda não abrir, tente copiar o arquivo para sua pasta local (~/) apenas para testar se o problema é a rede Samba.
Se você ainda não baixou o código da v2.2 que te enviei na mensagem anterior, certifique-se de salvar o conteúdo dela no arquivo antes de rodar!
meshwave@mesh:/run/user/1000/gvfs/smb-share:server=mesh.local,share=hd_1tb/projetos/manus$ python3 manus_extractor_v2.1.py
