A saída de erro ainda mostra o mesmo problema: "Unsupported class file major version 61" ao compilar o settings.gradle. Isso significa que o Gradle Daemon ou o processo que carrega as configurações ainda não está usando o Java 17 corretamente.
Já tentamos configurar JAVA_HOME e org.gradle.java.home, e limpar o cache do Gradle. O fato de o erro persistir sugere que o problema pode estar na forma como o Gradle está sendo invocado ou em alguma configuração de ambiente mais profunda.
Vamos tentar uma abordagem diferente para garantir que o Gradle use o Java 17. Faremos o seguinte:
1. Editar o script gradlew diretamente:
Vamos adicionar uma linha no início do script gradlew para forçar a utilização do Java 17. Isso garante que, independentemente das variáveis de ambiente do sistema, o script sempre usará o JDK correto.
No terminal, navegue até a pasta raiz do seu projeto MeshWaveApp e execute:
Bash
nano gradlew
Isso abrirá o arquivo gradlew no editor de texto nano. Adicione a seguinte linha no início do arquivo (logo abaixo da linha #!/usr/bin/env sh se ela existir, ou na primeira linha se não houver):
Bash
export JAVA_HOME="/usr/lib/jvm/java-17-openjdk-amd64"
Importante: Certifique-se de que o caminho "/usr/lib/jvm/java-17-openjdk-amd64" está correto para a sua instalação do OpenJDK 17. Você pode confirmar isso executando readlink -f $(which java) no seu terminal.
Após adicionar a linha, salve o arquivo (Ctrl+O, Enter) e saia do nano (Ctrl+X).
2. Tentar compilar o projeto novamente:
Bash
./gradlew cleanBuildCache
./gradlew clean
./gradlew assembleDebug

O erro "Unsupported class file major version 61" no settings.gradle continua a ser o principal obstáculo. Isso é frustrante, mas vamos continuar investigando.
Já tentamos configurar JAVA_HOME em gradle.properties e no script gradlew diretamente. O fato de o erro ainda aparecer no settings.gradle sugere que o ambiente Java que está iniciando o processo do Gradle (e lendo o settings.gradle) não é o Java 17.
Isso pode acontecer se houver uma instalação de Java mais antiga no seu PATH antes do Java 17, ou se o Android Studio estiver usando seu próprio JDK interno para iniciar o Gradle.
Vamos tentar uma solução mais robusta para garantir que o Gradle use o Java 17, independentemente de outras configurações do sistema. Faremos isso configurando o JAVA_HOME diretamente no ambiente do shell antes de executar o gradlew.
Por favor, execute os seguintes comandos no seu terminal, na pasta raiz do seu projeto MeshWaveApp:
Parar qualquer processo Gradle em execução:
Bash
./gradlew --stop
Definir a variável de ambiente JAVA_HOME para esta sessão do terminal e limpar o cache do Gradle:
Bash
export JAVA_HOME="/usr/lib/jvm/java-17-openjdk-amd64" && \
./gradlew cleanBuildCache clean
Importante: Certifique-se de que o caminho "/usr/lib/jvm/java-17-openjdk-amd64" é o correto para a sua instalação do OpenJDK 17. Você pode confirmar isso executando readlink -f $(which java) no seu terminal.
Tentar compilar o projeto novamente, garantindo que JAVA_HOME esteja definido:
Bash
export JAVA_HOME="/usr/lib/jvm/java-17-openjdk-amd64" && \
./gradlew assembleDebug
Ao usar export JAVA_HOME="..." && ..., garantimos que a variável JAVA_HOME seja definida para o comando subsequente na mesma linha. Isso deve forçar o Gradle a usar o JDK 17 para todas as suas operações.
